JavaSE 查漏补缺（二）

发表于 2019-07-24

枚举和内部类

枚举特征
  • 枚举类默认继承了java.lang.Enum,而不是object类。
  • 除了不能继承，基本上可以将enum 看做一个常规的类。
  • 成员内部类
  • 成员内部类可以无条件访问外部类的所有成员属性和成员方法（包括private成员和静态成员和方法）
  • 可以使用成员内部类实现多继承，除了该外围类，其他类都不能访问。
  • 成员内部类可以用private、protected、public、以及不写、来修饰类；而外部类自能被public、不写,修饰类。
  • 成员内部类中不能存在任何static的变量和方法。
局部内部类
  • 它和成员内部类的区别在于局部内部类的访问仅限于方法内或者该作用域内。
  • 局部内部类就像是方法里面的一个局部变量一样，是不能有访问修饰符以及static修饰符的。
匿名内部类
  • 1.4.1 匿名内部类不能定义任何静态成员、方法。
  • 1.4.2 匿名内部类中的方法不能是抽象的。(因为抽象类要被实现，不要没意义)
  • 1.4.3 匿名内部类必须实现接口或抽象父类的所有抽象方法。
  • 1.4.4 匿名内部类不能定义构造器。
  • 1.4.5 匿名内部类访问的外部类成员变量或成员方法必须用static修饰。
  • 1.4.6 内部类可以访问外部类私有变量和方法。
静态内部类
  • 只不过在类的前面多了一个关键字static
  • 它不能使用外部类的非static成员变量或者方法

异常

注意点
  • Try是必须的，Catch是可选的，Finally也是可选的；但必须二选一，不能只写Try。
  • 程序无法通过语法规则来判断业务的问题，就可以通过throw来自己根据业务抛异常。
  • 运行时异常可捕捉，非运行时异常不可捕捉。
自定义异常
  • 通过继承Exception来定义自己的异常类型;（构造器不能继承过来,他会自动帮你写一个无参的构造器）

集合

注意点
  • 集合和数组中存放的都是对象的引用。
  • Collection是集合类的接口，Collections是集合的工具类。
  • List 有序，可重复；Set 无序（无法判断他的插入后的顺序），不可重复；Map是以键值对存储的（也叫双集合存储，key集合通过指向找到value集合）。
  • 在对集合进行迭代过程中，不允许出现迭代器以外的对元素的操作，因为这样会产生安全隐患，java会抛出异常并发修改异常（ConcurrentModificationException），普通迭代器只支持在迭代过程中的删除动作。
List
  • ArrayList 查找速度快，插入操作慢(数组结构)。
  • LinkedList 插入速度较快，查询慢（双向链表结构）。
  • Vector: 描述的是一个线程安全的ArrayList。
  • List专属的迭代器listIterator();逆向迭代。
Set

  • 3.3.1 HashSet

    • HashSet不存入重复元素的规则.他会自动使用hashcode和equals进行比较。
    • 当存对象时一般地址都不一样，这时要判断他的值是否相同时就需要自己在对象类中重写hashcode和equals方法来判断是否相同（业务中常用）。

  • 3.3.2 TreeSet

    • TreeSet用的是红黑树算法：左小右大。
    • 存入的数据或对象一定要有排序规则，否则无法存入。
    • TreeSet集合排序的两种方式：（当Comparable比较方式和Comparator比较方式同时存在时，以Comparator的比较方式为主；）

  • 3.3.2 LinkedHashSet

    • 他会保存插入的顺序。
Map

  • 注意点：

    • Map集合没有直接取出元素的方法，而是先转成Set集合，在通过迭代获取元素 。
    • V put(K key, V value)：可以相同的key值，但是添加的value值会覆
      盖前面的，返回值是前一个的value，如果没有就返回null）。
    • Map的key是同过Set存储，value是通过Collection存储。
    • Map三种遍历方式

  • HashMap：

    • 底层是哈希表数据结构，线程是不同步的，可以存入null键，null值（因为key相同，所以无论存多少次null都会只有一个，value会覆盖旧的）。
    • 一般都会重写hashcode和equals方法。因为equals只判断存储的内存地址是否相同，业务上一般要判断对象的值是否相同。

  • TreeMap：

    • 和上面的TreeSet基本一致，不过TreeMap是要Key值要有比较性，实现和上面一致；

  • LinkedHashMap

    • 有序的HashMap存储方式（即可以按照插入顺序存储）

  • HashTable

    • 线程安全

    泛型

  • 泛型的特点：

    • 将运行时的异常提前至编译时发生。
    • 获取元素的时候无需强转类型，避免类型转换的异常问题。
    • 操作的引用数据类型不确定的时候,就可以使用泛型类。

  • 泛型类

    • 修饰符 class 类名<泛型>{};如 public class xie<K,T>{};
    • 可以定义一个或多个泛型类型，当调用他时必须赋给他类型；如 public class xie<String,int>{};
    • 子类继承他时，要么赋给他类型，要么和父类的泛型类型一致。
    • 接口也一样。

  • 泛型方法

    • Public < T > T fun(T t) {}

  • 泛型通配符

    • List< ? >：表示只能存List的子类或本类的数据，不能存父类和其他类的数据。
    • 可以对类型进行限定范围。
    • ？extends E: 接收E类型或者E的子类型。
    • ？super E: 接收E类型或者E的父类型。
